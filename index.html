Creature Clicker

In this game, a user tries to defeat monsters by clicking them. Each click is counted as an "attack" against the monster, and it loses health points accordingly. In order to win the player must defeat each monster before the timer (white bar) runs out.

This is a personal project that I completed after taking a few Java courses. I wanted to use my newly acquired knowledge about object-oriented programming to create something on a larger scale than the projects I had worked on in class. At the time, this was the largest and most complicated project that I had worked on, and I was very proud of the result (especially my use of polymorphism and inheritance). A friend created the art for the project, which I animated using timers.

  • Written in Java
  • Swing framework front-end
  • Java timers to animate art in real time
  • Embodies the four pillars of object-oriented programming

Game Of Life

This program is a representation of Conway's Game of Life. It is aptly described in this excerpt taken from wikipedia: "It is a zero-player game, meaning that its evolution is determined by its initial state, requiring no further input. One interacts with the Game of Life by creating an initial configuration and observing how it evolves. It is Turing complete and can simulate a universal constructor or any other Turing machine." The rules are simple:
  • Any live cell with two or three live neighbours survives.
  • Any dead cell with three live neighbours becomes a live cell.
  • All other live cells die in the next generation. Similarly, all other dead cells stay dead.
I've been fascinated with Conway's Game of Life since reading about it years ago. After taking some programming classes I realized that I could make the game myself, and couldn't help myself but code it up over a weekend.

  • Written in Java
  • Swing framework front-end
  • Uses timer to animates cells in real time
  • Embodies the four pillars of object oriented design
  • Click here to check it out on Github
